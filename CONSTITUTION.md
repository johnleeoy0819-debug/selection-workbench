# Maple Hollow Home — 项目原则

> 本文档定义项目的核心约束、决策标准和不可妥协的原则。
> 所有技术决策必须与此文档一致。

---

## 1. 项目定位

**产品名称：** Maple Hollow Home
**目标用户：** Etsy 实体产品卖家（1人团队，零经验起步）
**核心品类：** Home & Living（桌面装饰 + 墙面装饰）
**第一款产品：** 挂钩（Hook）
**价格带：** $15-60（基础款 $15-30，定制款 $30-60）
**目标市场：** 美国 / 英国 / 加拿大

---

## 2. 核心原则

### 2.1 数据驱动，人工决策
- AI 是分析助手，不是决策者
- 所有关键决策（入池、上架、放弃）必须人工确认
- 预测数据 vs 实际数据的偏差必须追踪和校准

### 2.2 合规优先
- Etsy 政策红线不可触碰
- 必须有设计参与（不能简单转售 1688 成品）
- 必须公开 Production Partner

### 2.3 快速验证
- 72 小时生死测试：上架后 72 小时内必须有曝光/点击数据
- 数据不好立即调整或放弃
- 不恋战，不执着

### 2.4 本地化优先
- 数据库本地 SQLite（可随时迁移）
- 前端静态部署（Vercel）
- 后端 EC2（18.234.173.5）

---

## 3. 技术约束

| 层级 | 技术 | 约束 |
|------|------|------|
| 前端 | React 19 + Vite 6 | 手写 CSS，不用 Tailwind |
| 后端 | FastAPI + Pydantic v2 | Python 3.12+ |
| 数据库 | SQLite | 本地文件，可迁移 |
| 采集 | OpenCLI | 用户本地运行，上传结果 |
| LLM | DeepSeek V4 Pro | 通过 OpenRouter |
| 部署 | Vercel + EC2 | 分离部署 |

---

## 4. 非目标（明确不做）

- ❌ 自动抓取 eRank（手动导出 CSV）
- ❌ 自动抓取 Etsy 前台（手动录入）
- ❌ 自动抓取 1688（OpenCLI 本地运行）
- ❌ 多店铺管理（v1.0 单店）
- ❌ 广告管理
- ❌ 库存同步
- ❌ 自动上架 Etsy

## 5. 决策标准

### 5.1 评分权重（可调整，需版本锁定）
| 维度 | 权重 | 说明 |
|------|------|------|
| 需求 | 35% | 月搜索量、趋势 |
| 利润 | 30% | 毛利率、净利 |
| 竞争 | 25% | 竞品数量、差异化空间 |
| 节日 | 10% | 季节性需求 |

### 5.2 决策阈值
| 综合评分 | 决策 | 说明 |
|---------|------|------|
| 80-100 | 立即执行 | 高潜力，优先上架 |
| 60-79 | 观察测试 | 有条件尝试 |
| <60 | 放弃 | 不浪费资源 |

---

## 6. 数据边界

### 6.1 Etsy API 限制
- ❌ 不获取 Shop Stats（Etsy 故意不开放）
- ✅ 可获取：店铺 Listings、Orders、Shipping Profile
- ✅ 可获取：Marketplace 级 Active Listings 搜索

### 6.2 汇率处理
- 默认 7.0 CNY/USD
- 支持 API 实时获取（失败时回退到默认值并告警）

### 6.3 Offsite Ads
- 年销售额 <$10K：0%
- 年销售额 ≥$10K：15%（自动检测）

---

## 7. 质量红线

- 所有 Python 函数必须有类型标注和 Docstring
- 所有 API 必须有输入验证（Pydantic）
- 所有用户输入必须防注入
- 无硬编码密钥（环境变量）
- 测试覆盖率 80%+
